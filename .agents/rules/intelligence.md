# 🧠 SOVEREIGN INTELLIGENCE (v13.0.0-PRODUCTION HARDENED)

## ⚡ MISSION STATEMENT
The system is governed by the **Enterprise Guardian**. Sovereign Intelligence enforces the alignment of activities with hardened outcomes, truth verification, and the 32-module complexity cap.

## ⚖️ INTELLIGENCE DIRECTIVES
1.  **Truth Consensus (v13.0.0-PRODUCTION)**: No internal metric is authoritative. The system MUST cross-verify SLIs from the [Performance Engine](../../scripts/performance-check.ps1) against infrastructure logs and governance state via the [Truth Bridge](../../scripts/truth.ps1).
2.  **Confidence-Based Autonomy**: Every autonomous turn MUST compute a Confidence Score.
    - **High (>80%)**: Ghost-Pilot / Full Autonomy.
    - **Medium (70-80%)**: Containment Mode (Restricted mutations).
    - **Low (<70%)**: **Autonomy Degradation**. The system MUST attempt an ultra-safe, minimal-risk path. If that path also yields low confidence, HALT and request Operator Override.
3.  **North-Star Goal Alignment**: The system MUST optimize for the **Stability-to-Complexity Ratio**. Busy activity (e.g., repeated linting/type-checking) that does not reduce architectural complexity is marked as "Misaligned Activity" and consumes double the Error Budget.
4.  **Outcome over Activity**: Success is defined by the reduction of high-priority warnings/findings in `sovereign-check.ps1`, not just the completion of an implementation plan.

## 🏛️ DECISION QUALITY
- **Irreversibility Filter**: Changes to core database schemas or supreme laws require a **Dual-Audit** signature (Two separate kluster review passes) before being committed to the authoritative state.
- **Tradeoff Logic**: The system MUST explicitly document in its metadata the logic used to select the current path over simpler alternatives.
